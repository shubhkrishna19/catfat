# Catfat Founder Discussion Notes

## What Is Already Stronger

- The storefront now has a clearer marketplace structure instead of looking like a broken export.
- Homepage, collection pages, and product pages are all rendering on the live dev preview.
- Product cards are now built from one shared component system instead of drifting section by section.
- Product pages now support:
  - clickable thumbnails
  - image switching
  - clearer variant selection
  - stronger price hierarchy
  - cleaner purchase flow
- The homepage now ends with trust and FAQ sections, which makes the store feel more complete and more credible.

## What Still Needs Founder Input

- Final brand voice for the homepage, footer, and reassurance copy.
- Which products should carry stronger merchandising tags:
  - bestseller
  - gifting pick
  - limited stock
  - bulk-order friendly
  - personalised
- Which collections should be pushed hardest in the top navigation and homepage.
- Whether the brand should lean more toward:
  - playful kids-first marketplace
  - premium lunchware gifting brand
  - discount-led high-conversion marketplace
- Final rules for coupon positioning, urgency language, and sale messaging.

## Real Catalog Issues Found

The theme is getting stronger, but catalog consistency is now becoming one of the main visible quality limits.

Examples from the live product sample:

- option names are inconsistent:
  - `Style`
  - `STYLE`
  - `THEME`
  - `STYLE 2`
  - `COLORS`
  - `COLOUR`
  - `COLOURS`
- some products have compare-at pricing and strong savings cues, others do not
- some handles are not easy to predict or are weak from a brand/SEO perspective:
  - `u-4`
  - `u-6`
  - `untitled-feb3_17-54`
  - `untitled-oct30_16-16`
- image counts vary a lot across products

## Recommended Founder Decisions

1. Approve a catalog normalization pass.
   - standardize option names
   - improve product handles
   - audit image order and featured images
   - clean product titles where needed

2. Approve a merchandising system.
   - define which badge types the store should use
   - define which products deserve them

3. Approve the primary conversion angle.
   - school utility
   - gifting
   - discount/value
   - kids character appeal

4. Approve the pre-launch QA scope.
   - how many products to test manually
   - which devices matter most
   - whether bulk-gifting flow needs dedicated review before launch

## Suggested Talking Points For The Meeting

- The design layer is now strong enough that catalog quality is the next real bottleneck.
- We should avoid publishing before catalog naming, images, and product messaging are cleaned.
- The current theme direction is working: colorful, sharper, more marketplace-led, less soft and generic.
- The next best ROI is not more random redesign. It is:
  - catalog cleanup
  - QA
  - final conversion polish
- We need to decide what Catfat is trying to win on first:
  - better visual browsing
  - gifting conversion
  - value-led sale conversion
  - character-driven kids demand

## Questions To Ask Founders

- Which 10 products should be treated as the hero products of the brand?
- Which 3 collections matter most for revenue?
- Do we want the brand to feel more premium or more marketplace-discount-heavy?
- Which products are truly personalised, and which are not?
- Should bulk gifting become a stronger top-level conversion path?
- Are there any products that should not be shown aggressively on the homepage?
